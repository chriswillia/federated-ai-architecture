Seller-only conversation guide

Trusted Federated AI battlecards

Use these as a shared field guide. Lead with the customer decision and open standards; use Microsoft products only as an illustrative anchor stack. Print for a meeting or co-sell planning session.

AI & apps seller

“Your teams will build agents wherever work happens. The decision is how to make that freedom reusable, measurable, and governed from day one.”

Discovery questions

  • Which teams are already building or buying agents, and where do they run?
  • Where are agents rebuilding the same integrations or knowledge sources?
  • Can you observe and inventory agents regardless of their build platform?
  • Which business outcome should own each agent’s cost and success measure?

Pillars owned

Experience and Build. Help teams choose the employee surface, builder experience, reusable prompts, tools, evaluations, and human approval points.

Example anchor stack

Copilot Studio, Microsoft Foundry, M365 Copilot, Foundry IQ, Microsoft Agents Framework — interchangeable with the customer’s chosen equivalent where the standards and controls remain intact.

Top objection

“We have already standardized on another framework.” Keep it. The value is the shared contracts around MCP, A2A, evaluation, inventory, and identity — not replacing a productive team’s framework.

CTA / next-step ladder

1. Map five active agents and their outcomes. 2. Choose one reusable MCP or API-gateway pattern. 3. Run a governed pilot. 4. Establish a portfolio review and scaling path.

Security seller

“You already govern people, apps, endpoints, and data. The next question is whether agents inherit that operating model or create an unobserved exception.”

Discovery questions

  • Do agents have named owners, workload identities, and defined permissions?
  • Can you apply access, DLP, and retention policy across clouds and SaaS?
  • Can an investigator trace prompt, tool call, data access, action, and output end to end?
  • What is your escalation model when an agent takes a high-impact action?
  • Where is unmanaged or shadow AI already visible?

Pillars owned

Govern and Secure. Lead identity, policy, observability, incident response, risk tiers, and lifecycle. Connect is a shared responsibility with the AI seller.

Example anchor stack

Entra, Purview, Sentinel, Defender, Agent 365, Foundry control-plane services — replaceable with equivalent identity, policy, SIEM/XDR, catalog, and runtime controls.

Top objection

“One control plane is just a Microsoft pitch.” Agree that a single console is not the goal. The goal is consistent identity, evidence, policy, and telemetry across vendor runtimes; Microsoft is one possible native anchor where it fits.

CTA / next-step ladder

1. Inventory one high-risk agent path. 2. Validate OAuth/OIDC scopes, DLP, and OpenTelemetry evidence. 3. Define tiered controls and an incident playbook. 4. Extend the pattern to the next zone.

Co-sell choreography

AI seller owns: Experience + BuildCustomer outcome, builder choices, agent lifecycle, evals, adoption, and outcome economics.
Security seller owns: Govern + SecureIdentity, policy, DLP, investigation, risk tiers, lifecycle controls, and incident response.
Shared: ConnectMCP, A2A, OAuth/OIDC, OpenTelemetry, and API-gateway design. Agree the portable contracts before proposing the anchor products.

Buyer-persona plays

CISO / SecurityLead with: “Can you see and investigate every consequential agent action?” Focus on shadow AI, workload identity, DLP, prompt injection, and incident response. CTA: AI security architecture review.
CIO / Enterprise architectLead with: “How do you prevent decentralized innovation from becoming decentralized technical debt?” Focus on reusable MCP/A2A/API patterns, inventory, and portability. CTA: federated architecture workshop.
Head of AI / AppsLead with: “How do teams ship useful agents without waiting for a central platform team?” Focus on builder enablement, evaluation, reuse, and portfolio management. CTA: governed pilot design.
Business sponsorLead with: “Which workflow should be materially faster, safer, or less expensive next quarter?” Focus on adoption, cycle time, and cost per outcome. CTA: select an outcome-backed scenario.
Joint pilot definition of doneBusiness outcome proven, identity and data controls validated, observable traces available, accountable owner named, and an expansion decision agreed.
Co-sell checklistAgree the customer problem, map the decision owner, select the reference scenario, assign AI/Security seller roles, and document the next workshop before leaving the meeting.
