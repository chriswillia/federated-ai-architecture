The framework

Centralize the controls, not the innovation.

Use the platforms that fit each team and workload. Apply a single model for identity, policy, evidence, and cost across all of them.

Seller journey

Lead with the customer framework. Pivot to the Microsoft anchor when a control gap is clear.

The site keeps the customer conversation vendor-neutral first, then helps AI and Security sellers jointly map an agreed control need to an example Microsoft architecture.

1 · Frame

Start with the federated reality

Agree that innovation will occur across platforms; the decision is how identity, policy, evidence, and cost remain consistent.

 2 · Diagnose

Find the first unmanaged boundary

Use the lifecycle to identify whether the immediate gap is discovery, identity, data, enforcement, or evidence.

 3 · Anchor

Pivot to the Agent 365 architecture

Show how Agent 365 can bridge discovery to managed identity and lifecycle, with Purview, Defender, and Sentinel enforcing the security model.

 4 · Advance

Agree the next workshop

Use the AI and Security seller plays to define the scenario, owner, control evidence, and outcome measure.
