Engineering Copilot (code → PR)

A development agent that reads a Jira ticket, grounds in the repo and tests via MCP, drafts a diff, runs pre-merge evals, and opens a PR for a human to approve — with a complete audit trail.

By the end of this flow, a developer goes from a Jira ticket to a reviewable PR with diffs, passing tests, and an eval scorecard — without leaving VS Code, and without anything shipping unreviewed.

A developer asks GitHub Copilot in VS Code to implement a Jira ticket. The engineering agent — built once in Foundry and versioned centrally — reads the ticket and the repo through MCP servers, drafts a diff, runs tests in a sandboxed build MCP, and submits the result for pre-merge Foundry evals (regression risk, OWASP weaknesses, license check). It opens a GitHub PR with the diff, test results, and an eval scorecard. A senior engineer reviews and merges — the agent never ships unreviewed. Every prompt, diff, eval, and approval lands in Purview for audit and the next eval cycle.

Reference architecture · Components by layer

A dev agent grounded in repo, tickets, and tests — with evals gating the merge.

The flow

Ticket → MCP-grounded read of repo + tests → plan & draft diff → sandboxed test run → Foundry evals → PR for human review → logged for audit and next eval.

What success looks like

Faster delivery on routine work
Refactors, dependency bumps, and small features land in hours — developers spend their time on architecture and review, not boilerplate.
Quality stays high
Pre-merge evals catch regressions, security weaknesses, and license issues before review — reviewers see the score, not the surprise.
Tooling stays sane
The agent only calls signed MCP servers from the central catalog — no shadow plugins, no rogue endpoints, no surprise data exfil.
Audit-ready by default
Every prompt, diff, test result, eval, and reviewer decision is in Purview — the SOX/SOC2 evidence is just a query.

Considerations

Eval suites are the gate
Pre-merge evals are only as good as the suite. Plan dedicated owners to maintain regression sets, security checks, and license rules in Foundry.
Human review is non-negotiable
No auto-merge. The agent drafts; a senior engineer ships. This is both a quality and an OWASP LLM01 (Prompt Injection) mitigation.
Sandbox the build MCP
Test execution must run in an isolated, ephemeral environment — no prod credentials, no shared state, no outbound network beyond allowlist.
OWASP LLM02 (Sensitive Disclosure)
Scope repo MCP access to the developer’s permissioned scope. Redact secrets in logs. Block uploads of customer data into prompts.
