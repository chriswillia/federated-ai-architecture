Cost-aware Expense Agent

Per-token bill-back, model routing for cheap vs. complex paths, and showback dashboards in Teams.

By the end of this flow, an employee submits a compliant expense in seconds — token cost stays capped per agent, every action is traceable, and Finance sees real-time showback by department.

An employee submits expenses in M365 Copilot (Teams). A Finance-owned Copilot Studio agent routes simple OCR work to a small model and escalates disputes to a larger one. Every token is tagged to Finance, policy is checked against Foundry IQ, calls to SAP S/4HANA and the corporate-card system are governed through Azure API Management, and Sentinel logs the chain end-to-end.

The flow

Receipts in, model-routed parsing, policy checks, SAP posting, and per-employee cost tagging.

What success looks like

Token spend, attributed
Each expense submission tags tokens to employee, cost center, and agent — Finance sees who and what is driving cost.
Lower per-receipt cost
Routing cheap OCR work to a small model can cut model spend 50–80% while still using a larger model for disputed items.
SOX-compliant audit trail
Sentinel logs every step; Purview records data movement — auditable in the same controls Finance already uses.
Faster approval cycle
Policy checks and approver routing happen in the agent — reimbursement time goes from days to hours.

Considerations

Model routing rules need tuning
Choosing when to escalate to a larger model is a real product decision. Expect 4–8 weeks of evaluation in Foundry to get it right.
Showback dashboards are bespoke today
Foundry Control Plane gives raw token telemetry. A polished per-employee/cost-center showback UI is still custom build.
Disputed expenses → human in loop
Don't auto-approve disputes. Always route to a human and log the decision; it's both a policy and OWASP LLM06 (Sensitive Disclosure) mitigation.
OWASP LLM10 (Unbounded Consumption)
Cap tokens per user per day, alert on anomalies in Sentinel, and rate-limit the agent at API Management to prevent runaway spend.
