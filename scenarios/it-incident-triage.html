Autonomous IT Incident Triage

An agent that classifies, diagnoses, and resolves Tier-1 incidents on its own — with signed runbooks, security A2A handoff, and human-in-the-loop escalation when confidence drops.

By the end of this flow, most P2/P3 incidents close themselves in minutes — with a complete, auditable action trail in Sentinel and a clean handoff to humans (or to the SOC) the moment something doesn’t fit the runbook.

An alert fires in Sentinel and opens a P2 incident in ServiceNow ITSM. Now Assist routes it to a Foundry-hosted triage agent. The agent classifies severity with a small model, escalates complex cases to a larger reasoning model, and only ever calls signed runbook tools from the central catalog. Diagnostics flow through Azure API Management; suspected security events hand off to a Defender agent via Agent2Agent; everything lands in one Sentinel trace. If confidence drops below threshold, the agent pages the on-call engineer with a complete summary.

Reference architecture · Components by layer

An autonomous agent — bounded by a signed tool catalog and a tight escalation policy.

The flow

Alert → classify → reason → act through signed tools → A2A handoff to the SOC if needed → HITL when confidence drops → post-mortem record.

What success looks like

Most Tier-1 tickets self-resolve
Pattern incidents (password resets, common service restarts, known runbook scenarios) close in minutes without an operator in the loop.
MTTR drops sharply on complex incidents
When humans are needed, they get a summary, a hypothesis, and the diagnostics already done — not a blank screen and an alert.
Security gets a fast lane
Anything that looks like an attacker exits ITSM and lands in Defender via A2A — in seconds, not after a manual escalation queue.
Every action is auditable
Sentinel captures every tool call, model decision, and outcome. The CMDB stays clean for the post-mortem.

Considerations

Tool catalog is the security boundary
The agent can only do what your signed runbook tools let it do. Treat the catalog like prod code — review, sign, and monitor it.
Escalation threshold needs tuning
Set the confidence threshold too high and humans get paged for everything; too low and the agent muddles through. Plan to iterate weekly for the first 90 days.
OWASP LLM07 (Insecure Plugin Design)
Every runbook tool needs least-privilege auth, parameter validation, and dry-run mode. Workload identities, not shared service accounts.
A2A trust requires Agent Cards
The Defender hand-off only works if both sides publish signed Agent Cards through the central registry. No registry, no A2A.
