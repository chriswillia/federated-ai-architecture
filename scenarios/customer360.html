Multi-source Customer 360 (Cross-vendor agent collaboration)

SAP Joule, Salesforce Agentforce, and a Foundry-built central agent working together as one.

By the end of this flow, a sales rep sees a single unified view across CRM, ERP, and support — stitched together by agents collaborating cross-vendor, with permissions and lineage preserved end-to-end.

A sales rep asks Salesforce Agentforce for a complete view of an account. Behind the scenes, Agentforce uses Agent2Agent (A2A) to discover and delegate to your Foundry-built Customer 360 agent and SAP Joule. They collaborate via Agent Cards, pull from Salesforce, SAP S/4HANA, M365 Graph, and Snowflake, ground in Foundry IQ, and return a unified view with lineage tracked in Purview.

Reference architecture · Components by layer

Cross-vendor agent collaboration relies on shared protocols (A2A, MCP), federated identity, and one place to see lineage.

The flow

Three agents from three vendors collaborate to answer a single sales-rep question.

What success looks like

360° view in seconds, not weeks
Sales rep gets a unified account picture without manually pivoting between Salesforce, SAP, and Snowflake dashboards.
No vendor lock-in
Vendor-neutral protocols (A2A, MCP) mean the central agent can collaborate with whatever SaaS you adopt next.
Data stays where it lives
No copy-out to a central lake. Each agent queries its own system and returns artifacts the collaborator can use.
One audit trail
Purview lineage and Foundry Control Plane traces show which agent touched which data and why.

Considerations

A2A is still maturing
Agent Card discovery, capability negotiation, and task lifecycle are early. Expect version churn and bespoke fallbacks for the first year.
Cross-vendor workload identity
Federating Entra, Salesforce, SAP IAS for agent-to-agent calls is harder than human SSO. Plan tokens, scopes, and rotation early.
Per-agent cost attribution gets tricky
When SAP Joule calls your Foundry agent which calls Agentforce, who pays which tokens? Tag conventions need to cross vendor boundaries.
Latency budget
Each hop adds 100–500ms. For real-time sales conversations, cache aggressively and design parallel A2A calls where you can.
