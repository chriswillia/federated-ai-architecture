Concepts

Eight ideas that make a federated AI operating model real.

The reference architecture shows the what. These concepts give sellers and architects the language to make it operational — inventory, controls, economics, and lifecycle for an estate of agents that will be built across many platforms and run in many clouds.

Concept 1

AI System of Record

Every enterprise has a system of record for people (HRIS), customers (CRM), finance (ERP), devices (endpoint), and identity (directory). Few have one for AI.

The agent registry becomes the AI system of record — the single authoritative inventory of every agent in the enterprise. It answers six questions:

  • What AI exists?
  • Who owns it?
  • What does it do?
  • What can it access?
  • What is its risk classification?
  • What value does it produce?

No agent should reach production without an entry in the AI system of record.

Concept 2

Agent Bill of Materials (ABOM)

Borrow the SBOM idea. Every production agent ships with a structured manifest that security, compliance, and business leaders can all read.

An ABOM includes:

  • Agent name, owner, sponsor
  • Platform, model, tools, data sources, APIs
  • Permissions and connected systems
  • Runtime environment
  • Cost profile and risk profile
  • Compliance requirements and evaluation results

The ABOM gives security, compliance, and business leaders a common language for an agent.

Concept 3

Agent Risk Tiers

Not every agent needs the same controls. Tiered classification makes governance practical rather than bureaucratic.

  • Tier 0 — Personal productivity agent. Low risk, user-scoped, no high-impact actions.
  • Tier 1 — Knowledge retrieval agent. Reads approved content, produces summaries.
  • Tier 2 — Workflow agent. Takes actions in business systems.
  • Tier 3 — Autonomous business process agent. Runs processes, uses tools, affects records.
  • Tier 4 — High-risk regulated agent. Touches sensitive data and regulated processes — financial, legal, HR, security, or customer-impacting decisions.

Controls scale with tier. Tier 0 ships fast; Tier 4 requires the full governance and security stack.

Concept 4

Trust Contracts

Every agent should ship with a trust contract — an explicit, machine-readable statement of what it can and cannot do.

A trust contract defines:

  • Who owns the agent
  • What the agent is allowed to do
  • What tools it can call and what data it can use
  • What outputs it can create
  • What approval is required
  • What logs must be captured
  • When access must be reviewed
  • When the agent should be retired

A production agent without a trust contract is unmanaged automation.

Concept 5

AI FinOps by Outcome

Token billing is the input metric. Move conversations to the output metric — the business outcome the agent produces.

Measure:

  • Cost per successful task
  • Cost per business process
  • Cost per avoided manual hour
  • Cost per resolved case
  • Cost per generated document
  • Cost per saved support ticket
  • Cost per reduced risk event

Token cost is the input metric. Business outcome is the value metric.

Concept 6

Progressive Governance

Heavy controls everywhere kill innovation. Progressive governance adds controls as an agent matures, so velocity and rigor scale together.

Six stages:

  • Experiment. Fast builder experience.
  • Register. Basic inventory and owner.
  • Observe. Telemetry, usage, cost, and behavior.
  • Govern. Policy, approval, access, lifecycle.
  • Secure. Identity, DLP, runtime protection, risk response.
  • Optimize. ROI, cost tuning, reuse, decommissioning.

Governance should increase as business impact and risk increase — not as a flat tax on every agent.

Concept 7

Agent Portfolio Management

Executives do not want 10,000 random agents. They want a managed portfolio they can steer like any other investment.

Portfolio views the operating model should expose:

  • Strategic agents and high-value agents
  • Redundant agents and abandoned agents
  • High-cost agents and high-risk agents
  • Experimental agents and production agents
  • Regulated agents

The future is not just building agents. It is managing an agent portfolio.

Concept 8

Federated AI Zones

The enterprise AI estate is zoned. The framework should not pretend every zone is governed equally today — it should show maturity by zone.

  • Zone 1 — Microsoft-native AI. M365 Copilot, Copilot Studio, Foundry.
  • Zone 2 — SaaS-native AI. SAP, ServiceNow, Salesforce, Workday, and other vendor copilots.
  • Zone 3 — Cloud-native AI. AWS, Google, Databricks, Snowflake, and other hyperscaler / data-platform AI.
  • Zone 4 — Custom AI. Internal apps, pro-code agents, custom APIs.
  • Zone 5 — Local / endpoint AI. Developer tools, local agents, browser agents, unmanaged tools.

The enterprise AI estate is zoned. The control plane must understand those zones.
